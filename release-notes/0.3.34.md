## What's changed

- copy the root package manifest into the managed OpenKit kit so Node sees the `type: module` boundary and no longer emits `MODULE_TYPELESS_PACKAGE_JSON` warnings for managed-kit ESM files
- include `src/cli` in the managed kit so runtime profile-switch imports resolve correctly after global materialization
- add regression coverage for managed-kit package metadata and materialized runtime import completeness

## Validation

- `node --test tests/global/ensure-install.test.js`
- `node --test tests/cli/profiles.test.js tests/runtime/profile-switch.test.js tests/runtime/launcher.test.js tests/cli/openkit-cli.test.js`
- `npm run verify:install-bundle`
