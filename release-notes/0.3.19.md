## What's changed

- add `openkit configure-embedding` CLI command to set up the embedding provider for semantic code search
- wire end-to-end embedding flow: `EmbeddingIndexer` auto-runs on file index, `SessionMemoryManager` embeds queries, `tool.semantic-search` prefers vector search and falls back to keyword search when embeddings are unavailable
- add `tool.embedding-index` runtime tool for operator-triggered indexing (index-file, index-project, status)
- add `src/global/embedding-config.js` helpers for reading, writing, clearing, and validating the `embedding` section of `.opencode/openkit.runtime.jsonc`
- add `commands/configure-embedding.md` operator docs with provider table, option reference, and recommended flow
- add 18-test CLI suite `tests/cli/configure-embedding.test.js` covering flag writes, merges, validation errors, clear, and interactive flows
- update README with `configure-embedding` quick-start section and supported providers table

## Notes

- embedding indexing is opt-in: set `embedding.enabled = true` in `.opencode/openkit.runtime.jsonc` or use `openkit configure-embedding --enable`
- restart `openkit run` after changing embedding config; config is read at bootstrap
- prefer `OPENAI_API_KEY` / `OLLAMA_HOST` env vars over storing keys in config when possible
