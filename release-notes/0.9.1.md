## What's changed

- **Defensive Error Handling**: Comprehensive hardening of project detection and config loading paths
- **Diagnostic System**: New structured logging system for debugging production issues
- **Zero Crashes**: Graceful fallbacks eliminate crashes on missing/invalid configs and unusual project structures

### Details

**New Diagnostic System:**
- Structured event logging to `.opencode/diagnostics.json`
- View with `openkit doctor --diagnostics`
- Automatic rotation (keeps last 1000 events)
- Categories: `config_loading`, `project_detection`, `runtime_bootstrap`, `mcp_init`, `capability_registry`

**Config Loading Hardening:**
- Three-layer defense: validation → graceful degradation → diagnostic logging
- Fallback chain: project config → user config → safe defaults (`loadRuntimeConfigWithDiagnostics`)
- Handle missing files, invalid JSON, wrong schema, permission errors
- New validators: `validateConfigFile`, `validateConfigSchema`, `parseRuntimeConfigContent`
- Never crashes, always returns a working config

**Project Detection Hardening:**
- Multi-strategy detection: cwd → walk up tree → project markers
- Handles nested directories (max 10 levels up the tree)
- Detects monorepos via workspace markers
- Resolves symlinks correctly
- New validators: `validateProjectRoot`, defensive root resolution
- Falls back to current directory instead of failing

**Edge Cases Handled:**
- Missing config files
- Invalid JSON/JSONC syntax
- Wrong config schema (non-object, missing `profiles.default`, wrong field types)
- Partial configs (additive merge with defaults)
- Corrupt config files
- Permission denied during read
- Nested project directories (invocation from deep subdirs)
- Symlinked directories
- Monorepo structures (workspace markers)
- No `package.json` (empty projects)

## Notes

- **Backward compatible**: No breaking changes; existing valid configs continue to load through the same fast path
- **Auto-recovery**: Invalid or missing configs now degrade to safe defaults instead of crashing; previously these surfaced as runtime errors
- **Diagnostic file**: New `.opencode/diagnostics.json` (gitignored) captures structured events for `openkit doctor --diagnostics`
- **Troubleshooting**: See `docs/operator/troubleshooting.md` and `docs/maintainer/error-codes.md`
- **Scope note**: The defensive primitives (validators, fallback chain, multi-strategy project detection) are landed as exported APIs and unit-tested end-to-end. The legacy strict `loadRuntimeConfig` path remains in place for callers that opt in via the new diagnostics variant; subsequent releases will widen the rollout.
