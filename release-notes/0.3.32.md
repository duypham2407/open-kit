## What's changed

- fix OpenCode v1.14.28 strict schema compatibility by removing OpenKit-only metadata from OpenCode-validated `opencode.json` files
- keep the default non-dangerous command allow behavior through the valid OpenCode `permission` projection
- preserve confirmation-required behavior for delete, destructive git, publish/release/deploy, database-destructive, and privileged/system-impacting commands
- update global/profile/install materialization, doctor drift detection, tests, and docs to derive policy health from `assets/default-command-permission-policy.json` instead of inline config metadata
