## What's changed

- Fixed session base-directory resolution so OpenKit consistently separates repository roots, managed workspace roots, and per-session runtime state.
- Fixed per-session workflow-state mirror handling to avoid nested `.opencode/sessions/.opencode/...` paths and to keep root compatibility mirrors in sync.
- Aligned quick-lane ownership and gate aliases with the live Quick Task+ contract, including `quick_intake` ownership and `quick_verified` approval normalization.
- Fixed retained-worktree launch rollback behavior so failed selection attempts do not accidentally restore a stale active work item.

## Validation

- `npm run verify:all`
- `git diff --check`
- Targeted session, governance, audit, launcher, workflow-state, and quick-lane regression tests passed during release preparation.

## Published package

- npm: `@duypham93/openkit@0.9.2`

## Notes

- Backward compatible: this release hardens runtime path resolution and workflow semantics without changing the public lane names or command surface.
- Operators using multi-session workflows should upgrade to pick up the corrected per-session mirror and launcher behavior.
