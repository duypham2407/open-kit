## What's changed

- move checked-in OpenKit authoring surfaces from the repository root into `src/kit` to avoid collisions with runtime-generated `.opencode` and compatibility shim paths
- retarget runtime loaders, workspace shims, install-bundle metadata, and session-start hooks so managed runtime behavior still works while source content now lives under `src/kit`
- refresh CLI, install, governance, runtime, and session-start regression coverage so package builds and managed wrappers stay aligned with the new source layout

- `npm run verify:mcp-secret-package-readiness`
- `npm run verify:install-bundle`

## Notes

- Root-level checked-in authoring directories such as `agents/`, `commands/`, `context/`, `skills/`, and `hooks/` are now packaged from `src/kit/`, while `.opencode` remains the runtime and compatibility surface.
- Managed workspace shims continue to expose compatibility paths for OpenCode and existing runtime helpers, but repository source-of-truth content should now be updated under `src/kit`.
