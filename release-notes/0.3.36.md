## What's changed

- add `openkit switch-profiles` as a direct interactive profile picker for the current OpenKit runtime session
- add `openkit switch` as a short alias for the direct profile switch command
- document the two available switch paths: `/switch-profiles` as the current OpenCode prompt-template slash command and `openkit switch-profiles` as the direct CLI picker
- keep session scoping fail-closed with `OPENKIT_RUNTIME_SESSION_ID` so direct switching cannot mutate the global default or other sessions

## Notes

- OpenCode custom slash-command files are currently prompt templates, not documented native executable commands. Use `openkit switch-profiles` or `openkit switch` when you want the direct picker path.
- Run `npm install -g @duypham93/openkit@latest`, then `openkit upgrade`, then restart `openkit run` to pick up the updated CLI and docs.
