## What's changed

OpenKit v0.6.0 is the **audit hardening release**. A structured audit of
the v0.5.1 surface (4 parallel read-only subagents + 5 cross-layer drift
checks) found 4 Critical, 15 High, 22 Medium, 13 Low issues plus 1
macOS-only post-audit bug. This release ships fixes for every Critical
and High and the majority of Medium/Low items.

### Highlights

- **No breaking API changes.** All fixes preserve existing public CLI /
  MCP / tool-input shapes. A few internals shifted behavior (gate system,
  workflow-kernel write methods, MCP `tool.workflow-state` schema) — see
  the **Behavior shifts** section if you integrate with workflow internals.
- **`npm run verify:all` is fully green for the first time in months.**
  The audit also surfaced 5 pre-existing test failures that were repaired
  during the baseline cleanup.
- **5 security boundaries closed**: shell injection in `tool.ast-grep-search`,
  config-driven RCE via `supervisorDialogue.openclaw.command` and external
  MCP stdio, env-var binary substitution via `OPENKIT_SECURITY_CLI`, and
  `file://` path traversal in the prompt loader.
- **Atomic global upgrade.** A mid-flight failure in
  `materializeGlobalInstall` no longer leaves the kit-root empty.
- **FSM consolidation.** Two divergent transition tables that disagreed on
  5 `full`/`migration` transitions are merged into a single canonical
  source. The merge chose the more permissive form, so backward-rework
  paths previously rejected silently are now allowed.

### Behavior shifts (worth re-verifying if you integrate with internals)

- **Gate system**: `gate-requirements.js` is now a thin shim over
  `GateRegistry`. Operators querying gates by the old
  `quick_intake→quick_plan` ID will see the gate is absent — it was
  semantically dead in the persistence layer. Gate IDs follow the
  `lane.snake_case` form going forward.
- **`tool.workflow-state` MCP schema**: drops the unused `command` enum
  and documents `workItemId` (the property the handler actually accepts).
  Models calling with `{ command: 'show' }` previously received a silent
  null; that input shape is no longer documented as supported.
- **Workflow-kernel write methods**: emit a stderr "needs-bootstrap" hint
  on a fresh project before returning null. Behavior unchanged after
  bootstrap; the hint just makes the no-op visible.
- **`switch-profiles` CLI**: now actually runs when spawned from a path
  under a symlinked prefix (notably macOS `/var/folders/...`). Previously
  exited 0 with empty stdout — silent no-op.

### Validation

- `npm run verify:all` — fully green
- `npm run verify:audit-wave-1` — 12/12 pass (regression suite for the
  four Critical fixes)
- Manual smoke: bootstrap each of the 3 lanes (quick / delivery /
  migrate) on a fresh project, verify advance-stage works
- Atomic-upgrade rollback test: synthetic mid-flight failure preserves
  the prior install (sentinel survives)
