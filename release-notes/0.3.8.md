## What's changed

- harden the global workflow-state bootstrap so workspace-backed wrappers hydrate from repo-local managed state instead of failing on empty active-work-item indexes
- pass `OPENKIT_PROJECT_ROOT` through generated workflow wrappers and normalize wrapper help aliases so `.opencode/openkit/workflow-state.js` behaves like the canonical root wrapper
- clarify the runtime path split so prompts and command docs treat `.opencode/work-items/` as canonical and `.opencode/openkit/work-items/` as compatibility-only
