## What's changed

- add MCP server bridge that exposes 27+ kit tools to OpenCode agents via the Model Context Protocol (stdio transport)
- kit tools (find-symbol, semantic-search, syntax-outline, import-graph, ast-grep-search, etc.) are now callable by the agent as real MCP tool calls, not just prompt references
- add JSON Schema input definitions for all 31 agent-facing tools across graph, syntax, AST, codemod, workflow, analysis, LSP, and external families
- configure `.opencode/opencode.json` with `mcp.openkit` local server declaration so OpenCode auto-connects on startup
- propagate MCP server config to global kit install via `materialize.js`
- update tool-substitution-rules.md to document MCP tool availability

## Notes

- the MCP server bootstraps the full OpenKit runtime in read-only mode; graph-dependent tools return 'unavailable' until the project is indexed (use tool.import-graph with action "index" to trigger indexing)
- tools that depend on runtime-internal managers (continuation, delegation, session, profile-switch) are intentionally excluded from MCP exposure
- requires `@modelcontextprotocol/sdk` ^1.29.0 (added as dependency)
