## What's changed

- normalize `profile-switch` runtime and wrapper responses so regular input mistakes return structured status objects instead of uncaught exceptions
- normalize syntax index path handling so invalid and missing files return stable runtime statuses rather than manager-level throws
- normalize managed and global launcher spawn failures into structured return payloads while preserving existing missing-`opencode` guidance
- expand regression coverage for wrapper output shape changes, syntax path errors, and launcher failure paths

## Notes

- this release continues the runtime hardening pass by reducing exception-oriented control flow in user-facing runtime surfaces
- existing successful wrapper and tool behavior is preserved; the main change is more predictable failure reporting for callers and automation
