## What's changed

- add managed AST tooling provisioning, syntax parsing, rule-audit, and codemod runtime surfaces to the OpenKit foundation
- introduce Semgrep-backed `rule-scan` and `security-scan` tools with bundled quality and security rule packs plus operator docs
- introduce jscodeshift-backed `codemod-preview` and `codemod-apply` tools, bundled sample codemods, doctor visibility, and preview-first guidance
- add model execution fallback state, dual quick-switch profiles, and the `/switch` command path for live runtime profile selection

## Validation

- `node --test tests/runtime/codemod-tools.test.js`
- `npm run verify:all`

## Published package

- npm: `@duypham93/openkit@0.3.14`

## Notes

- `node_modules/` is now ignored and remains excluded from the published release; consumers should install dependencies from `package.json` and `package-lock.json`
- codemod and audit tools report dependency or fallback state explicitly when their backing tooling is unavailable
