## What's changed

- **BREAKING:** removed `/task` command — users now pick a lane explicitly with `/quick-task`, `/delivery`, or `/migrate`
- **BREAKING:** removed `/brainstorm` command — brainstorm is now stage 0 inside each lane, owned by the first specialist agent (Quick Agent in `quick_plan`, Product Lead in `full_product`, Solution Lead in `migration_strategy`)
- fix the "no workflow" error class on fresh global installs — `workflow-state.json` is now created on the first command via the new `tool.bootstrap-workflow` MCP tool
- Master Orchestrator is now purely procedural: bootstraps state, dispatches the specialist, routes between stages, but no longer classifies lanes or owns domain content
- `*_intake` stages are now MasterOrchestrator-owned and ephemeral — MO bootstraps and advances immediately, never blocking for user input
- removed `quick_brainstorm` from the FSM (folded into `quick_plan`)
- brainstorm storage: quick lane writes a 50-100 word inline summary to `state.brainstorm`; full and migration lanes capture brainstorm in scope/migration plan files as Appendix A (discovery notes) and Appendix B (decisions)
- workspace-shim no longer crashes when the workspace state file does not yet exist on a fresh project
- `bootstrapWorkflow` records `lane_source = user_explicit` so escalation routing matches the contract
- expose `kernel.bootstrapWorkflow()` and `kernel.canWriteState()` on the workflow-kernel adapter
- add `bootstrap` subcommand to the `.opencode/workflow-state.js` CLI

## Notes

- **Breaking changes:** if you have scripts, agents, or muscle memory that uses `/task` or `/brainstorm`, switch to `/quick-task`, `/delivery`, or `/migrate`. Pick the lane based on the work shape — quick for bounded daily work, delivery for feature work that needs scope, migrate for upgrades that preserve behavior.
- The first command on a fresh project now bootstraps `workflow-state.json` automatically. No more manual `/task` to initialize.
- Migration lane currently runs `migration_baseline` before `migration_strategy` — Solution Lead's brainstorm in `migration_strategy` benefits from existing baseline evidence. A future change may reorder these stages so brainstorm shapes baseline directly.
- Run `npm install -g @duypham93/openkit@latest`, then `openkit upgrade`, then restart `openkit run` to pick up the new commands and bootstrap behavior.
