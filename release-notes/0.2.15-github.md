## Summary

- Add conditional parallel execution semantics for `Full Delivery` and `Migration`.
- Require plans to bless whether work stays sequential or can use bounded worker-pool parallelism.
- Add runtime support for full-delivery task allocation checks and migration slice boards.

## Highlights

- `parallelization` workflow-state metadata with `none`, `limited`, and `enabled` modes
- full-delivery task `concurrency_class` support
- `validate-task-allocation` and `integration-check` runtime helpers
- migration slice validation, ownership, and parity-oriented slice status flow
- new guidance for:
  - `docs/operations/runbooks/tech-lead-task-decomposition.md`
  - `docs/maintainer/conditional-parallel-execution-note.md`

## Validation

- `NODE_OPTIONS=--trace-warnings node --test`
- published as `@duypham93/openkit@0.2.15`

## Notes

- Parallel execution remains conditional and bounded, not automatic.
- Quick work stays non-parallel.
- Migration slices remain distinct from the full-delivery task board so lane semantics stay intact.
