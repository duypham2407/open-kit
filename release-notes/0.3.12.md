## What's changed

- teach `openkit doctor` to report when the global kit is missing required workflow templates such as `docs/templates/migration-solution-package-template.md`
- make the doctor guidance explicit that missing migration templates usually mean the global OpenKit install is stale or drifted and should be refreshed with `openkit upgrade`
- add regression coverage for the stale-global-kit migration-template diagnostic path

## Validation

- `node --test tests/global/doctor.test.js && node --test tests/runtime/doctor.test.js && npm run verify:install-bundle`
- `npm run verify:all`

## Published package

- npm: `@duypham93/openkit@0.3.12`

## Notes

- this release improves diagnosis only; it does not change migration lane semantics
- if `migrate` reports a missing migration template, `openkit doctor` should now point you toward `openkit upgrade`
