## What's changed

- enable `chrome-devtools-mcp` by default in both checked-in and managed OpenKit configs so browser testing/debugging tools are available out of the box
- allow managed install merge policy to add and preserve `mcp` config safely during install/upgrade materialization
- harden artifact scaffolding for full-delivery transitions by resolving templates from project and kit-root fallback paths (including managed compatibility layout)
- create missing scaffold output directories automatically instead of failing when `docs/scope` or `docs/solution` is absent
- extend install/global tests to assert default Chrome DevTools MCP behavior
