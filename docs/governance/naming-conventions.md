# Naming Conventions

## Artifact Filenames

- Use `YYYY-MM-DD-<slug>.md`
- Keep slugs lowercase and hyphen-separated
- Reuse the same slug across brief, spec, architecture, plan, and QA artifacts for one feature

## IDs

- Feature IDs use `FEATURE-###`
- ADR IDs use `ADR-###`
- QA issue IDs use `ISSUE-###`
